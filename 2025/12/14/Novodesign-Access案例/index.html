【实战案例】一句“天太专业了”背后：如何用Access打造企业级决策支持系统？

“天太专业了。。”

这是今天交付完项目后，客户发来的第一句反馈。
这不是一次普通的“代做作业”，而是一次从数据杂乱无章到自动化辅助决策的完整技术降维打击。

一、 客户的需求：不仅仅是“做个库”

这次的客户是一位面临结课作业压力的大学生。他的课题是为一家名为 Novodesign 的公司开发一套企业决策支持系统。

难点在于：老师要求的不仅仅是“存储数据”，而是要对数据进行深度整合与分析，要在关键决策场景（如选址、库存、生产）提供可视化的数据支持。

客户之前的状态：

  • 手握一堆 Excel 表格，数据分散。
  • 不知道怎么用 Access 实现复杂的业务逻辑（比如按重量自动分配货位）。
  • 最头疼的是：系统做出来也就算了，还要写一份专业的《用户使用说明书》，这对很多理工科学生来说简直是噩梦。

二、 站长的解决方案：Access + BI思维

接单后，我没有把它当成一个“作业”来做，而是把它当成一个微型ERP系统来架构。

我利用 Microsoft Access 的强大功能，为 Novodesign 定制了四大核心功能模块：

1. 智能仓库货位决策方案

这不是简单的记录东西放在哪。我在后台写入了逻辑判断算法：

  • 自动称重逻辑：系统自动识别 >70kg 的商品，强制标记为“重型货物区”，建议叉车搬运。
  • 频次分析逻辑：自动统计出库频率（>100次），将爆款商品自动分配到 A区（近出口），极大地缩短了搬运距离。

2. 科学的新建工厂选址分析

为了降低物流成本，我利用 SQL 聚合查询，对客户通讯录进行了地理维度分析：

  • 系统自动统计各州（State）的现有客户数量，并按降序排列。
  • 决策输出：直接生成报表，指出客户最密集的区域（如 CA 加利福尼亚州），为建厂选址提供硬数据支撑。

3. 产品产量决策支持

基于历史销量制定生产计划，避免库存积压：

  • 畅销品策略：针对头部商品（Occasional Chair等）生成增产建议。
  • 滞销品策略：针对尾部滑销品生成减产或停止生产建议。

三、 交付标准：不仅交付代码，更交付“文档”

很多开发者只管代码能跑，不管用户会不会用。但我认为，专业的文档是系统的一部分。

为了让客户能顺利通过答辩（并在老师面前秀一把），我为他编写了一份企业级的《用户使用说明书》。
